# 사용자 흐름 및 페이지 구조

## 핵심 사용자 흐름
회원가입/로그인 → 대시보드 진입 → OKR 생성 → 업무기록 페이지 이동 → 날짜 선택 → 업무기록 모달 작성 → TODO 입력 → OKR 여러 개 연결 → 저장 → 대시보드 확인

## 핵심 UX 규칙
- 업무기록 작성/수정은 모달로 처리한다.
- OKR 생성/수정도 모달로 처리한다.
- 캘린더는 라이브러리를 사용한다.
- 캘린더 셀에는 한줄 요약 일부만 노출한다.
- 긴 텍스트는 한 줄 `...` 처리한다.
- 저장은 항상 한 번에 전체 저장한다.
- 저장 실패 시 토스트만 노출하고 추가 복구 로직은 없다.
- archived OKR은 업무기록 연결 모달에서 선택할 수 없다.

## 페이지 구성
실제 페이지는 4개로 제한한다.

### 1) `/auth`
목적:
- 로그인 / 회원가입

핵심 UI:
- 이메일 입력
- 비밀번호 입력
- 로그인 버튼
- 회원가입 버튼
- 전환 링크
- 에러 메시지

### 2) `/dashboard`
목적:
- 전체 기록 현황과 목표 진행 상태를 빠르게 확인

핵심 UI:
- 기록된 일수 카드
- 활성 OKR 개수 카드
- 완료 OKR 개수 카드
- OKR 달성률 목록
- 상위 5개 표시 + `더보기` 클릭 시 전체 표시
- 오늘 할 일 목록
- 내일 할 일 미리보기
- 빠른 액션 버튼

표시 규칙:
- 오늘 할 일에는 완료된 TODO도 함께 표시
- 완료된 TODO는 체크 상태 유지
- archived OKR은 기본 진행 목록에서 제외

### 3) `/okrs`
목적:
- OKR CRUD

핵심 UI:
- OKR 목록
- `OKR 추가` 버튼
- 생성/수정 모달
- 삭제 확인 다이얼로그
- 상태 배지 또는 상태 선택 UI

OKR 입력 항목:
- 목표
- 목표수치
- 현재수치
- 단위
- 기간 유형
- 시작일
- 종료일
- 상태

### 4) `/work-logs`
목적:
- 날짜별 업무기록 관리

핵심 UI:
- 월간 캘린더
- 이전달 / 다음달 이동
- 오늘 기록 작성 버튼
- 날짜 셀에 기록 여부 표시
- 날짜 셀에 한줄 요약 일부 노출
- 날짜 클릭 시 작성/수정 모달 오픈

## 업무기록 모달 구성
### 기본
- 날짜
- 하루 한줄 요약
- 연관 OKR 멀티 선택

### TODO
- 오전 TODO 리스트
- 오후 TODO 리스트
- TODO 추가 / 삭제 / 완료 체크

### 회고 항목
- 한일
- 이슈/장애/버그
- 막힌 것
- 설계/의사결정
- 배운점
- 내일 할일
- 수치 변화
- 받은 피드백
- 개선 포인트

## 빈 상태 문구
### OKR 없음
- 등록된 OKR이 없어요.
- 첫 목표를 만들어보세요.

### 이번 달 업무기록 없음
- 이번 달에는 아직 업무기록이 없어요.
- 날짜를 클릭해서 첫 기록을 남겨보세요.

### 대시보드에 오늘 기록 없음
- 오늘 작성된 업무기록이 없어요.
- 오늘 기록 작성 버튼 제공
